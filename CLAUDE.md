# 贝叶斯公式交互演示

一个单文件 HTML 演示页面，用「面积 = 概率」的二维面积图（eikosogram）帮助理解贝叶斯公式。适合教学、自学或嵌入到更大课程中作为交互组件。

## 已实现功能

- **面积图直接操控三个原生参数**（无隐藏约束，整张正方形永远合法）
  - 拖动蓝色竖直分割线：调整先验 `P(H)`（左列宽度）
  - 在左列内上下拖动：调整似然 `P(D|H)`（绿区上沿）
  - 在右列内上下拖动：调整反假设似然 `P(D|¬H)`（红区上沿）
  - `P(D)` 与后验 `P(H|D)` 作为推导结果实时显示，标注「推导」徽章
- **场景预设**
  - 🏥 医学检测：`P(H)=0.01`、`P(D|H)=0.9`、`P(D|¬H)=0.1` → 后验仅 ≈8.3%，演示基础率谬误
  - 📧 垃圾邮件：`P(H)=0.4`、`P(D|H)=0.6`、`P(D|¬H)=0.05` → 后验 ≈89%
  - ✏️ 自定义：手动拖动任意参数自动切换；每个场景配情境说明句（H/D 的具体含义）
- **自然频数视图（每 1000 个个体）**
  - 50×20 点阵（waffle）：绿 = H 真且出现 D，红 = H 假但出现 D，浅蓝 = H 真但未出现 D，灰 = 其余
  - 图例实时显示四类计数；下方文字按场景拆解「阳性者中真正 H 的比例 = 后验」
- **公式实时展示**
  - 符号行：`P(H|D) = P(D|H)·P(H) / P(D)`，附 `P(D)` 全展开
  - 数字代入行：随参数实时代入，如 `0.9 · 0.01 / 0.108 ≈ 0.083`（最多三位小数、去尾零）
- **动态指标与后验条**
  - 实时显示 `P(H)`、`P(D|H)`、`P(D|¬H)`、`P(D)`、`P(H|D)`
  - 紫色渐变后验条（紫色 = 后验，与公式的颜色语义一致）
  - 情境化解读文案：基础率谬误提示、无判别力（l≈f）、决定性证据（f≈0）等按优先级给出
- **分享与状态**
  - 参数序列化到 URL hash（`#p=0.01&l=0.9&f=0.1`，防抖 250ms），刷新/分享可还原
  - 「🔗 分享」按钮一键复制链接
- **辅助控制与可访问性**
  - 明暗主题切换，`localStorage` 持久化，默认跟随系统偏好
  - 三个手柄均为 `role="slider"` 可聚焦元素：方向键 ±0.01，Shift+方向键 ±0.1，Home/End 到端点，带 `aria-valuenow`
  - 点阵 SVG 带 `aria-label` 计数摘要；解读与场景说明 `aria-live="polite"`
  - `prefers-reduced-motion` 下禁用过渡动画
  - 响应式布局，适配窄屏

## 如何友好地演示贝叶斯公式

### 核心教学叙事

1. **先建立直觉**：先验 `P(H)` 是看到数据前对假设的相信程度（左列宽度）。
2. **引入证据**：`P(D|H)` 是假设成立时数据出现的可能性（绿区高度）；`P(D|¬H)` 是假设不成立时数据出现的可能性（红区高度）。
3. **面积即联合概率**：绿区面积 = `P(H)·P(D|H)`（真证据），红区面积 = `P(¬H)·P(D|¬H)`（伪证据），两者之和 = `P(D)`。
4. **得出后验**：后验 = 绿区面积占绿+红总面积的比例 —— 「支持 H 的证据」在「所有证据」中的占比。
5. **用自然频数巩固**：切到 1000 人点阵，数一数绿格与红格，后验就是 绿/(绿+红)，与公式结果互相印证。

### 演示时可以强调的互动点

- **从医学检测预设开场**：患病率 1%、检测敏感性 90%、假阳性率 10%，问学生"阳性者真患病的概率？"——多数人猜 90%，实际仅 ≈8.3%。这是贝叶斯最震撼的一课。
- **把 `P(D|¬H)` 拖向 0**：红区消失，后验飙升——证据若几乎不可能由反假设解释，一次观察就足以定论。
- **让 `P(D|H)` ≈ `P(D|¬H)`**：绿红高度相同，后验 ≈ 先验——没有判别力的证据不改变任何判断。
- **把 `P(D|H)` 与 `P(D|¬H)` 都拖到 0**：`P(D)=0`，后验无定义（显示「—」）——不可能出现的数据无法作为条件。
- **先验极低但证据极强**（如 p=0.01, l=0.99, f=0.001）：低基础率也能被有力证据推翻。

### 可视化映射

| 视觉元素 | 数学含义 |
|---------|---------|
| 左列（蓝）宽度 | `P(H)` |
| 右列（灰）宽度 | `P(¬H)` |
| 绿色区域面积 | `P(H)·P(D|H)`（分子，真证据） |
| 红色区域面积 | `P(¬H)·P(D|¬H)`（伪证据） |
| 绿+红区域面积 | `P(D)` |
| 绿色面积 / (绿+红) | `P(H|D)` |
| 点阵绿格数 / (绿+红格数) | 后验的自然频数表达 |

## 技术栈

- **纯前端**：单个 HTML 文件，无构建工具、无依赖
- **SVG**：面积图与 1000 格点阵（点阵仅对变色格子做增量更新）
- **原生 JavaScript (ES6+)**：Pointer Events 统一鼠标/触摸，`setPointerCapture` 拖拽
- **CSS 变量**：统一控制浅色/深色主题和配色
- **无框架**：便于直接嵌入任何静态站点或课程页面

## 界面展示风格

### 设计原则

- **清晰优先**：公式、指标、图表都用颜色编码一一对应
- **颜色语义固定**：
  - 蓝色 = 先验 `P(H)`（分割线、手柄、公式标签）
  - 绿色 = 似然 `P(D|H)` 与真证据区域
  - 红色 = 反假设似然 `P(D|¬H)` 与伪证据区域
  - 青色 = 数据概率 `P(D)`
  - 紫色 = 后验 `P(H|D)`（公式标签与后验条）
- **卡片式布局**：左侧面积图，右侧公式与指标，底部通栏自然频数
- **轻量阴影与圆角**：现代、友好的视觉感受
- **明暗主题**：默认跟随系统偏好，可手动切换并持久化
- **无障碍考虑**：
  - 手柄可 Tab 聚焦、方向键操作，focus 有琥珀色描边
  - 区域标签使用背景药丸避免与彩色区域混淆
  - 图表装饰元素 `pointer-events="none"`，拖拽命中由统一命中测试处理

### 响应式

- 桌面：左右双栏 + 底部通栏
- 平板/手机（≤820px）：单栏堆叠，顶部按钮改为流式居中
- SVG 使用 `viewBox`，可随容器自动缩放

## 数学模型说明

页面内部状态（三个原生参数，均由用户直接控制）：

- `prior` = `P(H)`，拖动蓝色分割线或 ←/→ 方向键，范围 `[0, 1]`
- `likelihood` = `P(D|H)`，左列内竖拖或绿手柄 ↑/↓ 方向键，范围 `[0, 1]`
- `falseLikelihood` = `P(D|¬H)`，右列内竖拖或红手柄 ↑/↓ 方向键，范围 `[0, 1]`

推导量：

```
P(D)    = P(H)·P(D|H) + P(¬H)·P(D|¬H)
P(H|D)  = P(H)·P(D|H) / P(D)        （P(D) > 0 时）
```

后验就是绿区面积占绿红总面积的比例，也是点阵中 绿格/(绿格+红格)。

## 注意事项

- 当 `P(D) = 0`（如两个似然均为 0）时，`P(H|D)` 无定义：页面显示「—」，后验条清空，并给出专门解读文案。
- 数字显示最多三位小数并去尾零（`0.9`、`0.099`、`0.083`），后验条使用百分比整数。
- 点阵计数对各类四舍五入，并以灰色组吸收舍入误差，保证总数恒为 1000。
- 拖拽命中优先级：手柄（16px）> 分割线（±14px）> 整列区域，窄列下手柄依然可抓取。
- 主题通过 `data-theme` 属性与 CSS 变量实现，用户选择持久化到 `localStorage`（键 `bayes-theme`）。
- URL hash 在拖动停止 250ms 后写入（`replaceState`，不污染历史记录）；加载时解析并 clamp 到 `[0,1]`。
